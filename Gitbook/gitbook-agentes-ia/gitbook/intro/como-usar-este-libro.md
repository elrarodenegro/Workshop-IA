# Cómo usar este libro

## Las convenciones que vas a ver

Para que no pierdas el ritmo, este libro usa convenciones consistentes en todos los capítulos.

### Bloques de código

Cuando veas un bloque como este:

```python
print("Hola, agente")
```

Significa que es código que vas a escribir en tu computador. Si tiene un encabezado como `# archivo: agent.py`, te indica el nombre del archivo donde va.

### Comandos de terminal

Cuando veas un bloque con `$` al inicio:

```bash
$ python --version
```

Es un comando que vas a escribir en tu terminal. **No copies el `$`** — solo lo que viene después.

### Callouts (avisos)

Verás cuatro tipos de avisos a lo largo del libro:

> 💡 **Tip**
>
> Información que te ahorra tiempo o errores.

> ⚠️ **Cuidado**
>
> Algo que suele salir mal si no prestas atención.

> 📌 **Nota**
>
> Contexto adicional que vale la pena saber pero no es crítico.

> 🎯 **Reto**
>
> Un ejercicio opcional para practicar lo aprendido.

## Cómo seguir el ritmo

### Si eres principiante total

Lee cada parte en orden. **No saltes la Parte 0** — sin el setup, los casos del workshop no van a funcionar y vas a frustrarte. Mejor invertir 30 minutos al inicio que dos horas debugueando después.

### Si ya tienes Python instalado

Salta directo a la **Parte 0.5** (crear API keys) y luego a la Parte 1. Si los conceptos los tienes claros, vete derecho al **Caso 1** del workshop.

### Si solo quieres la teoría

Lee Partes 1 y 2. Son independientes del workshop y te dan la base conceptual completa.

### Si solo quieres construir

Setup (Parte 0) → Workshop (Parte 3). Vuelves a los conceptos cuando algo no te cuadre.

## Velocidad sugerida

Esto no es una carrera, pero si quieres una referencia:

| Si dispones de... | Plan sugerido |
|-------------------|---------------|
| **1 fin de semana** | Setup + Parte 1 + Caso 1 |
| **1 semana (1h/día)** | Todo el libro completo |
| **2-3 horas seguidas** | Setup + Caso 1 (es el único 100% sin código) |

## Cuando algo no funcione

**Lo primero que va a pasar es que algo no te funcione.** Es normal. Es la parte más educativa.

1. **Lee el mensaje de error completo.** No solo la última línea — el error suele estar en la primera o segunda línea del traceback.
2. **Busca el error en el [Apéndice D · FAQ y errores frecuentes](../apendices/d-faq.md).**
3. **Pregúntale al agente** (sí, irónicamente). Pega el error a Claude o ChatGPT y di "estoy siguiendo este tutorial y me sale este error". Va a explicarte qué pasa el 90% de las veces.
4. **Si nada funciona**, vuelve atrás un paso. Casi siempre el error está en el paso anterior, no en el actual.
